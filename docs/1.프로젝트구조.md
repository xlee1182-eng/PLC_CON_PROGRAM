📋 프로젝트 개요
PLC(산업용 제어기) 통신 시스템으로, FastAPI 기반의 비동기 웹 서버입니다. 여러 종류의 PLC 드라이버를 지원하며 HTTP API를 통해 PLC와 통신합니다.

▶️ 실행 절차 순서 (중요!)
1단계: 서버 시작
    python server.py
            ↓
    uvicorn.run() 호출
    - host: 0.0.0.0
    - port: 5000

2단계: 설정 로드 (AppConfig.SET)
    Config.json 읽기
        ↓
    APPCONFIG 딕셔너리 생성
    {
        'PLC_INFO': {
        'OPCUA': {...},
        'MITSUBISHI': {...},
        ...
        }
    }
            ↓
    활성화된 PLC 정보 로그 출력

3단계: FastAPI 앱 초기화 (main.py)
    FastAPI 인스턴스 생성
            ↓
    CORS 미들웨어 추가 (모든 출처 허용)
            ↓
    정적 파일 마운트 (/static → app/web/)
            ↓
    라우터 등록 (RootApi.py)

4단계: 서버 시작 시 lifespan (비동기 컨텍스트)
    lifespan() 함수 호출
            ↓
    asyncio.create_task(__JOB_PLC.START())
    └─→ 백그라운드에서 PLC 초기화 작업 시작

5단계: PLC 초기화 (plcjob.START)
    Config.json에서 'USEYN': 'Y' 인 PLC만 처리
            ↓
    각 PLC 타입별로 driver_factory.create_driver() 호출
    ├─ OPCUA  → AsyncOPCUAPLC 생성
    ├─ OPCDA  → AsyncOPCDA 생성
    ├─ MITSUBISHI → AsyncMitsubishiPLC 생성
    ├─ SIEMENS → AsyncSiemensPLC 생성
    └─ ROCKWELL → AsyncRockwellPLC 생성
            ↓
    PLC 인스턴스들을 리스트에 저장
            ↓
    AsyncPLCManager(plc_list) 생성
            ↓
    PLC_MANAGER.start() 실행 (각 PLC 연결 시작)
            ↓
    logger.info() 로 시작 완료 메시지 출력

6단계: 웹 서버 구동
    포트 5000에서 리스닝 시작
            ↓
    클라이언트 요청 수락 준비 완료

7단계: API 요청 처리
    주요 엔드포인트:
    메서드	URL	설명
    GET	/	서버 상태 확인
    GET	/web	대시보드 UI (dashboard.html)
    GET	/plc_read?plc_name=XXX&tag=YYY	PLC 태그 값 읽기
    GET	/plc_write?plc_name=XXX&tag=YYY&value=ZZZ	PLC 태그에 값 쓰기
    POST	/api/plc/read-tag	JSON으로 PLC 읽기
    POST	/api/plc/write-tag	JSON으로 PLC 쓰기
    GET	/plc_list	연결된 PLC 목록 조회

    API 요청 흐름 예시:
    GET /plc_read?plc_name=MITSU_PLC_1&tag=D100
            ↓
    RootApi.plc_read() 함수 실행
            ↓
    PLC_MANAGER.read_by_name('MITSU_PLC_1', 'D100') 호출
            ↓
    AsyncPLCManager에서 해당 PLC 검색
            ↓
    AsyncMitsubishiPLC.read_tag('D100') 실행
            ↓
    실제 PLC와 통신 (비동기)
            ↓
    값 반환 → JSON 응답 (RESPONSEFORMAT)

8단계: 서버 종료
    Ctrl+C 신호 수신
            ↓
    lifespan 컨텍스트 종료
            ↓
    PLC_MANAGER.stop() 실행 (모든 PLC 연결 해제)
            ↓
    백그라운드 task 취소
            ↓
    서버 완전 종료

🔄 의존성 관계도
server.py
    ↓
AppConfig.SET() ← Config.json
    ↓
main.py (FastAPI 앱)
    ├─ CORS 미들웨어
    ├─ 정적 파일 (web/)
    ├─ lifespan (비동기 초기화)
    │   └─ plcjob.START()
    │       ├─ driver_factory.create_driver()
    │       │   ├─ opcua_async.AsyncOPCUAPLC
    │       │   ├─ mitsubishi_async.AsyncMitsubishiPLC
    │       │   ├─ siemens_async.AsyncSiemensPLC
    │       │   └─ ...
    │       └─ plc_manager.AsyncPLCManager.start()
    │
    └─ routes/index.py
        └─ api/RootApi.py (HTTP 엔드포인트)
            └─ plcjob.GET_PLC_MANAGER()
                └─ AsyncPLCManager.read_by_name/write_by_name

CommonFunction.RESPONSEFORMAT() ← 모든 API 응답 포매팅

🚀 실행 명령
# 1. 의존성 설치
pip install -r requirements.txt

# 2. Config.json 설정 (IP, PORT, TAGS 수정)
# 3. 서버 실행
python server.py

# 4. 웹 브라우저에서 접속
http://localhost:5000/web

# 5. API 테스트 (curl 예시)
curl "http://localhost:5000/plc_read?plc_name=MITSU_PLC_1&tag=D100"

🔧 새로운 PLC 드라이버 추가 절차
1.plc_drivers 에 새로운 파일 생성 (예: example_async.py)
2.BasePLC 클래스 상속 후 드라이버 구현
3.driver_factory.py 에서:
    빌더 함수 작성 (_build_example)
    DRIVER_BUILDERS 딕셔너리에 등록
4.Config.json 에서 새 PLC 타입 설정
5.USEYN: "Y" 로 활성화
