OPC DA와 OPC UA는 산업 자동화에서 PLC 데이터를 외부 시스템으로 전달하기 위한 표준 통신 기술입니다.
📘 1. OPC란?

OPC는 Open Platform Communications의 약자로,
서로 다른 장비(PLC, HMI, 서버 등) 간 데이터를 표준 방식으로 교환하기 위한 기술입니다.

📘 2. OPC DA (Data Access)
✔ 개요

1990년대 후반 등장

초기 OPC 표준

Windows 전용 기술

✔ 특징

Microsoft COM/DCOM 기반

실시간 데이터 읽기/쓰기 지원

같은 PC 또는 내부 네트워크에서 주로 사용

✔ 구조
[ PLC ] ←→ [ OPC DA Server ] ←→ [ OPC DA Client ]
                (Windows COM/DCOM)
✔ 장점

초기 산업 자동화에서 널리 사용

구현된 시스템 많음 (레거시)

❌ 단점 (중요)

OS 의존 (Windows만 가능)

DCOM 설정 매우 복잡

방화벽/네트워크 문제 많음

보안 취약 (암호화 없음)

👉 요약: 과거 기술, 유지보수용

📘 3. OPC UA (Unified Architecture)
✔ 개요

OPC DA의 문제 해결을 위해 등장

최신 OPC 표준

현재 산업 표준

✔ 특징

플랫폼 독립 (Windows, Linux, 임베디드)

TCP/IP 기반

보안 내장 (암호화, 인증)

데이터 구조 모델링 지원

✔ 구조
[ PLC ] ←→ [ OPC UA Server ] ←→ [ OPC UA Client ]
              (TCP/IP + 보안)
✔ 장점

✔ OS 독립
✔ 보안 (SSL/TLS)
✔ 인터넷/클라우드 연동 가능
✔ 확장성 뛰어남
✔ 구조화된 데이터 모델 지원

✔ 추가 기능

Subscription (데이터 변경 시 push)

Event / Alarm 처리

복잡한 데이터 구조 표현 가능

📘 4. OPC DA vs OPC UA 비교
항목	OPC DA	OPC UA
출시 시기	1990s	2008 이후
플랫폼	Windows 전용	플랫폼 독립
통신 방식	COM/DCOM	TCP/IP
보안	없음	있음 (암호화/인증)
설정 난이도	매우 어려움	비교적 쉬움
확장성	낮음	매우 높음
추천 여부	❌ 신규 비추천	✅ 필수
📘 5. 데이터 처리 방식 차이
OPC DA

Polling 중심 (주기적으로 읽기)

OPC UA

Polling + Subscription 지원

👉 Subscription 예시

값 변경 시 자동 알림 (이벤트 기반)
